BUGS FOUND + FIXED ALONG THE WAY (on develop branch)
================================================================================
1. src/utils/response.ts had two back-to-back modules stuck together, causing a
   syntax error + duplicate `created` export. Merged into one module; added
   `sendSuccess` alias (controllers were importing a name that didn't exist).
2. src/routes/index.ts line 11 had `app.use(...)` at module top level
   referencing undefined `app`. Replaced with a clean router registry mounting
   all 20 routes.
3. src/server.ts imported `connectDatabase` / `disconnectDatabase` from
   config/database that never existed. Added both helpers.
4. src/models/wallet.model.ts returned NUMERIC balances as strings (pg default)
   despite `balance: number` declared type. Normalized via `Number()`.
5. src/services/tracking.service.ts assertSessionOwner did `SELECT *`
   (snake_case) then accessed `session.providerId` (camelCase) — always
   undefined, so every updateLocation/endSession call returned 403. Fixed by
   aliasing columns.
6. src/controllers/tracking.controller.ts used `const { sessionId } =
   req.params` — under @types/express 5 this is `string | string[]` and fails
   strict typecheck. Cast to string.
7. src/middleware/errorHandler.ts called `logger.error({ obj }, "msg")` (pino
   style) but the logger is winston which wants `(msg, obj)`. Swapped args.
8. src/middleware/errorHandler.ts regex `/^\d{5}$/` did not match Postgres
   SQLSTATE 22P02 (has a letter). Broadened to `/^[0-9A-Z]{5}$/`.
9. src/middleware/errorHandler.ts only recognized `AppError` instances;
   services across the codebase throw plain `Error` objects with `statusCode`
   and `code` attached via `Object.assign`. Now normalized into AppError.
10. src/middleware/validate.ts called `schema.parseAsync(...)` but never wrote
    the result back to req.body/params/query, so Zod transforms
    (`.trim()`, `.toUpperCase()`, defaults) were silently dropped. Now applies
    transformed values back onto the request.
11. src/middleware/auth.ts attached `req.user` with shape `{ sub, role, phone }`
    but every controller uses `req.user.id`. Added an `id` alias (= sub) in the
    payload so old + new code shape both work.
